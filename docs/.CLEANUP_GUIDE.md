```python

@app.get("/proyecto/{id}", tags=["Almacenamiento"])
async def get_proyecto(id: str):
    """Recupera un proyecto guardado de Firestore."""
    try:
        firestore_client = FirestoreClient()
        project_data = await firestore_client.get_project(id)
        if not project_data:
            raise HTTPException(status_code=404, detail="Proyecto no encontrado")
        return project_data
    except HTTPException:
        raise
    except Exception as e:
        logger.error("Error recuperando proyecto %s: %s", id, e)
        raise HTTPException(status_code=503, detail="Error recuperando proyecto")
```

**Acción:** Eliminar completamente. Si lo necesitas en v2, está en el historio git.

## 🟡 PRIORIDAD 1 — Limpiar Dependencias

### 1. Eliminar `google-cloud-secret-manager` de requirements.txt

**Ubicación:** `backend/requirements.txt` línea ~11

```
# ❌ ELIMINAR
google-cloud-secret-manager==2.20.0
```

**Por qué:**
- Cloud Run maneja esto nativamente a través de ADC (Application Default Credentials)
- El código nunca importa `google.cloud.secretmanager`
- En Cloud Run, `GOOGLE_API_KEY` viene inyectado por Secret Manager sin que el código lo pida

**Verificación:**
```bash
# Grep para confirmación
grep -r "secretmanager" backend/
# Resultado: (ninguno)

grep -r "SECRET" backend/
# Solo sale en comentarios, no en importes
```

### 2. Considerar lazy-loading de componentes

Si el bundle crece, considera hacer dinámicos estos componentes:

```tsx
// frontend/app/page.tsx
import dynamic from 'next/dynamic';

const CodeView = dynamic(() => import('@/components/CodeView'), {
  loading: () => <p>Cargando código...</p>,
});

const RationaleView = dynamic(() => import('@/components/RationaleView'), {
  loading: () => <p>Cargando rationale...</p>,
});
```

Esto reduce el JS inicial y carga bajo demanda.
